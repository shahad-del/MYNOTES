Here’s the command to start the console, logging in as the MySQL database administrator (which uses the root user ID):

mysql -u root -p

Here’s the console command that creates our database: 

mysql> create database vsearchlogDB;

Here’s the SQL statement we used to create the required table (called log). 

mysql> create table log

This next command creates a new MySQL user called vsearch, uses “vsearchpasswd” as the new user’s password, and gives the vsearch user full rights to the vsearchlogDB database: 

CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON * . * TO 'newuser'@'localhost';

The asterisks in this command refer to the database and table (respectively) that they can access.

Once you have finalized the permissions that you want to set up for your new users, always be sure to reload all the privileges.

FLUSH PRIVILEGES;

To provide a specific user with a permission, you can use this framework: 

GRANT type_of_permission ON database_name.table_name TO 'username'@'localhost';

If you want to give them access to any database or to any table, make sure to put an asterisk (*) in the place of the database name or table name. Each time you update or change a permission be sure to use the Flush Privileges command. If you need to revoke a permission, the structure is almost identical to granting it:

REVOKE type_of_permission ON database_name.table_name FROM 'username'@'localhost';

Note that when revoking permissions, the syntax requires that you use FROM, instead of TO as we used when granting permissions. You can review a user’s current permissions by running the following:

SHOW GRANTS FOR 'username'@'localhost';

Just as you can delete databases with DROP, you can use DROP to delete a user altogether:

DROP USER 'username'@'localhost';

The simplest thing is to cancel the query. However, you cannot just type \c in this case, because mysql interprets it as part of the string that it is collecting. Instead, enter the closing quote character (so mysql knows you've finished the string), then type \c: 

mysql> SELECT * FROM my_table WHERE name = 'Smith AND age < 30;
        '> '\c

The prompt changes back to mysql>, indicating that mysql is ready for a new query. The `> prompt is similar to the '> and "> prompts, but indicates that you have begun but not completed a back-tick-quoted identifier.It is important to know what the '>, ">, and `> prompts signify, because if you mistakenly enter an unterminated string, any further lines you type appear to be ignored by mysql—including a line containing QUIT. This can be quite confusing, especially if you do not know that you need to supply the terminating quote before you can cancel the current query.

GROUP_BY AND HAVING

  1. HAVING function comes after GROUP BY FUNCTION and allows us to filter after aggregation has already taken place.Aggregatiion takes place after group by statement,that's why we can't use it with WHERE clause as it takes place before GROUP BY FUNCTION.To filter out the data on statements based on agregated functions like SUM(amount),we have to use HAVING and that also after GROUP BY FUNCTION.Filter with WHERE statement on SUM amount is not possible.

    2. 
SELECT customer_id,sum(amount) FROM payment
GROUP BY customer_id
HAVING SUM(amount) > 100

  2. GROUP BY comes after from clause or where clause,not after ORDER BY or others.
  3. GROUP BY should includes all those columns which are included in select statement without using agregate function like MAX,AVG,COUNT.
  4. GROUP BY function is not called by agregate functions. 

    5. select customer_id, count(amount),sum(amount)FROM payment
        GROUP BY customer_id
        ORDER BY customer_id;
If there is some other column also after customer_id,it should be in GROUP BY call too but not count(amount),sum(amount) as they are agregate function.

JOINS

Finally join statement which let us query between two or more tables based on something that joins it.

Inner Join
Inner join produces only the set of records that match in both Table A and Table B.

SELECT DISTINCT(title) FROM film inner join inventory ON film.film_id = inventory.film_id ORDER BY title;
-- DOUBLE INNER JOIN SYNTAX
SELECT title,first_name,last_name FROM film_actor INNER JOIN actor ON film_actor.actor_id= actor.actor_id INNER JOIN film ON film_actor.film_id = film.film_id WHERE first_name = 'Nick' and last_name = 'Wahlberg';
Full outer join
Full outer join produces the set of all records in Table A and Table B, with matching records from both sides where available getting repeated.We use it too when we want to have unique records in both tables that is not available in either of them using WHERE.

SELECT * FROM TableB FULL OUTER JOIN TableA ON TableA.col_match = TableA.col_match where TableA.id is null OR TableB.id is null
Left outer join
Left outer join produces a complete set of records from Table A, with only the matching records (where available) in Table B.To produce the set of records only in Table A, but not in Table B, we perform the same left outer join.Order matters in left outer join in contrast to others discussed above.if we want to have record of data available in Table A, but not in Table B,since left outer join joins all the data present in Table A and only the common one from Table B,we can use WHERE clause.The result of this query has a rows in TableB containing null records which means that, those records are not available in TableA,that's why their result is Null.We can use this NULL to query for null records.

SELECT *FROM TableA LEFT JOIN TableB ON TableA.col_match = TableA.column match WHERE TableB.id is null.

SELF-JOIN allows us to join different column of the same table.Alias is important in self-join as we are taking columns of same table,so different name is necessary to differentiate between them.

SELECT movie.title,next.title,movie.length FROM film as movie join film as next ON

Constraints

Constraints are rules enforced on tables and columns. These are used to prevent invalid data being entered into the database.Two categories of it are:column constraints and table constraints.Most common constraints are:

column constraints

  1. NOT NULL
  2. UNIQUE
  3. primary key
  4. foreign key
  5. check constraint
  6. Exclusion

Table constraints

  1. check
  2. references
  3. unique
  4. primary key

Primary key is unique to the table where as through foreign key we connect to other tables.While using forign key we use key word 'REFERENCES' and has to make sure that the value is same as the table to which it is referencing.

syntax of check constraint:

CREATE Table employees(
emp_id SERIAL PRIMARY KEY,
first_name varchar(50) NOT NULL UNIQUE,
last_name varchar(50) NOT NULL,
birthdate DATE check(birthdate > '1900-01-01'),
hire_date DATE check (hire_date > birthdate),
salary INTEGER check(salary > 0)
)

Condtional expressions and procedures

We can use CASE statements only when we have certain conditions.It is like if and else statements.There are two main ways of using CASE statements,either using general CASE OR through CASE expression.General case allows all kind of comparisons like between,greater than and many more.

Syntax for general CASE is:

select column_name,
CASE
    WHEN condition1 THEN  result1
    WHEN condition2 THEN  result2
    ELSE some_other _result
    END FROM TABLE_NAME
SELECT WORKER,
CASE 
    WHEN WORKER = 'JONES' THEN '1ST'
    CASE WHEN WORKER = 'JACK' THEN '2ND'
    ELSE 'OTHER'
    END FROM COMPANY;
select customer_id,
case 
    when customer_id < 100 THEN 'PREMIUM'
    WHEN customer_id BETWEEN 100 AND 200 THEN 'PLUS'
    ELSE 'NORMAL'
    END AS Customer_Category
    from customer;    

Unlike general Case,Case expression  checks only for one condition like equal to.

Syntax for CASE expression:


SELECT column_name,
CASE expression
    WHEN value1 THEN  result1
    WHEN value2 THEN  result2
    ELSE some_other_result
    END FROM TABLE_NAME;
SELECT WORKER,
CASE WORKER 
    WHEN 1 THEN 'JONES'
    WHEN 2 THEN 'JACK'
    ELSE 'OTHER'
    END FROM COMPANY;
SELECT 
 SUM(case rental_rate
    when 0.99 THEN 1
    ELSE 0
    END) as bargains,
 SUM(case rental_rate
    when 4.99 THEN 1
    ELSE 0
    END) as premium
    from film

COALESCE FUNCTION

COALESCE FUNCTION accepts unlimited number of arguments.It returns the first argument that is not null and if all arguments are null then the function will return null.It becomes useful when we are querying a table that contains null values and want to substitute with another value.For example we have table of products and want to find out the final cost after discount have been given but there is one column where the amount of discount is not mentioned .In such cases coaslesce function is useful and makes us easier to work with NULL values. 


SELECT item,(price - COALESCE(discount,0))

CAST

CAST operator let us convert from one data type to another but we have to keep in mind that all data type cannot be CAST to another data type.For example it is reasonable to convert string '5' to integer 5 not string'five'.Though it is more appropriate to use CAST in a column name instead of a single instance as said in the example.


SQL CAST SYNTAX
SELECT CAST('5' AS INTEGER)
select CHAR_LENGTH(cast(inventory_id AS VARCHAR)) from rental;
POSTGRESQL CAST SYNTAXE
SELECT '5':: INTEGER
select CHAR_LENGTH(inventory_id :: VARCHAR )from rental;

View

Often we have to keep querying complex combination of tables and conditions again and again.Instead of performing same query again and again ,we can create a VIEW and just use it instead of writing all those syntaxes again.VIEW doesn't store data physically,it simply stores the query. 


create view view_example AS
SELECT first_name,last_name,address FROM customer 
INNER JOIN address
ON customer.address_id = address.address_id
select * from view_example
